## 2. 현재 프로젝트 상태

현재 프로젝트는 아래 2가지 저장소 모드를 지원함.

  - 브라우저 `localStorage` 사용

현재 `.env`에는 아래 값이 들어가 있으며, 앱은 Supabase 모드로 동작하도록 설정되어 있음.

```env
VITE_APP_DATA_PROVIDER=supabase
VITE_SUPABASE_URL=https://jcjskudlcgqsrapsmhvk.supabase.co
VITE_SUPABASE_ANON_KEY=sb_publishable_rI-zGFJ1-l1bhSAzTarMPg_xndvqfv5
```

주의:
- `.env`는 `.gitignore`에 포함되어 있음
- Git 추적되지 않음

## 6. Supabase 저장 로직 현재 방식

현재 저장 로직은 `portfolio_key = 'default'` 한 건을 기준으로 동작함.

저장 순서:

1. 먼저 `update` 시도
2. 업데이트된 행이 없으면 `insert` 시도
3. 만약 insert에서 unique 충돌 발생 시 다시 `update` 재시도

의도:

- 중복 insert 가능성 최소화
- 단일 snapshot 구조 유지
- 충돌 시 최대한 안전하게 회복

관련 파일:

- `src/repositories/supabasePortfolioRepository.ts`

## 11. 주의사항

- `.env`는 절대 커밋하지 않기
- 현재 Supabase anon key는 로컬 테스트 기준으로만 사용 중
- `default` snapshot 구조는 MVP에는 적합하지만 장기적으로는 holdings / transactions / notes 분리 검토 필요
- REST 테스트 중 `default` payload를 한 차례 갱신했고, 작업 종료 전에 샘플 포트폴리오 payload로 복구 완료
