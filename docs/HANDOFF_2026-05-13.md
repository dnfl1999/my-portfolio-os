## 8. 실제 앱 오류 대응 추가 기록

사용자가 실제 앱에서 데이터가 반영되지 않고 오류가 난다고 제보함.

최종 원인:

- 앱이 바라보는 Supabase 도메인 `jcjskudlcgqsrapsmhvk.supabase.co`가 외부 DNS에서도 `Non-existent domain`으로 확인됨
- GitHub Pages 배포 번들에는 해당 Supabase URL과 anon key가 포함되어 있었음
- 따라서 앱 오류의 직접 원인은 가격 API가 아니라 Supabase 저장소 로드 실패였음

응급 조치:

- `src/data/mockData.ts`
  - `scripts/portfolio-default-payload.json`을 fallback 기본 포트폴리오로 import
  - Supabase가 죽어도 빈 배열 대신 실제 보유종목 payload를 앱에서 볼 수 있게 함
- `src/hooks/usePortfolioStore.ts`
  - 최초 저장소 로드 실패 시 `mockPortfolioData`를 앱 상태로 넣고 상단 저장 오류 배너는 띄우지 않도록 조정
  - 저장/동기화 자체는 Supabase 복구 전까지 여전히 실패할 수 있음
- 기존 stale state 보정도 유지
  - `usePortfolioStore.ts`: 최신 `latestDataRef` 기준으로 현재가 snapshot 반영
  - `useLivePrices.ts`: 최신 holdings/provider ref 기준으로 현재가 요청
