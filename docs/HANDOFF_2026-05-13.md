## 3. 새로 발견한 블로커

- 현재 환경에서 Supabase 프로젝트 도메인 `jcjskudlcgqsrapsmhvk.supabase.co`가 DNS 해석되지 않음
- 직접 REST 확인도 실패:
  - `The remote name could not be resolved: 'jcjskudlcgqsrapsmhvk.supabase.co'`
- headless Chrome으로 배포 앱을 열면 화면은 렌더링되지만 저장소 로드가 실패함
  - 콘솔: `Supabase 데이터 로드에 실패했습니다: TypeError: Failed to fetch`
- 따라서 실제 Supabase snapshot을 읽은 상태에서의 배포 화면 검증은 오늘 환경에서는 막힘
- 우선 Supabase 프로젝트가 pause/deleted 상태인지, 또는 프로젝트 URL/secret이 바뀌었는지 Supabase 대시보드에서 확인 필요

## 4. 오늘 수정한 코드

- `src/hooks/usePortfolioStore.ts`
  - 가격 snapshot 적용 시 오래된 `data` 클로저 대신 최신 포트폴리오 ref를 기준으로 반영하도록 수정
  - 빠르게 종목을 추가/수정한 직후 현재가 갱신 응답이 늦게 오면 최신 holdings를 덮어쓸 수 있던 위험을 줄임
- `src/hooks/useLivePrices.ts`
  - 현재가 조회 시 최신 holdings/provider ref를 사용하도록 수정
  - in-flight 요청이 있을 때 중복 요청을 막는 기준을 `isRefreshing` 클로저 대신 `inFlightRef`로 정리
